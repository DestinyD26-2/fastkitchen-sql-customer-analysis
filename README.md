FastKitchen Customer SQL Analysis

📊 Overview
This project analyzes customer and order data for a fictional restaurant to understand user behavior and spending patterns.

🎯 Objective
To compare registered and guest users and identify opportunities to improve customer retention.

🛠️ Tools Used
- SQL
- Dataset: FastKitchen Orders & Users

🔍 Key Questions
- How much do customers spend on average?
- Are registered users or guests more active?
- How does spending vary by location?

📈 Key Insights
- Average order value is ~$22
- Guest users place more orders than registered users
- Some zip codes have higher average spending than others
- Registration friction may reduce user sign-ups

📂 Project Structure
- `queries.sql`
- `insights.md`

## 🚀 Skills Demonstrated
- Joins (LEFT JOIN)
- Aggregations (AVG, COUNT)
- Filtering with WHERE
- Grouping and sorting

## 💡 Business Recommendation
Simplify account registration and offer incentives to convert guest users into registered customers.
